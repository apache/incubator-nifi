Description

The VaultHttpClientControllerService manages an HTTP or HTTPS client connection to a Hashicorp Vault server, and provides key-value store access to it.

Configuration Details

URL

The HTTP or HTTPS URL of the Vault Server. Typically similar to https://127.0.0.1:8200.

Secrets Path

The path to the KV secrets engine. Typically kv or similar. See KV Secrets Engine for further details.

Key Value Store Prefix

The path prefix used for storing values by the controller. Default is /, which means the service will store properties at the root.

Auth Type

The authentication type the controller will use when connecting and interacting with the Vault Server.

Token

The value of the authentication to send when using "Token Authentication". See Token Auth Method for details.

App ID

The value of the "app id" to send when using "App Id Authentication". See AppId Auth Method for details.

User Id Mechanism

Specifies how the "user id" will be created before sending it during "App Id Authentication". The controller service currently supports "Static User Id", and "IP Address" mechanisms. See AppId Auth Method for details.

Role Id

The value of the "role id" to send when using "App Role Authentication". See AppRole Auth Method for details.

Secret Id

The value of the "secret id" to send when using "App Role Authentication". See AppRole Auth Method for details.
